A contribuição escolhida foi ajudar a resolver as issues [Dataset Duplication Metrics & Mapping Export #13](https://github.com/linux-duks/MailingListsHeritage/issues/13) e [Replication Analytics & Visualizations #14](https://github.com/linux-duks/MailingListsHeritage/issues/14) do repositório [MLH-archiver](https://github.com/linux-duks/MLH-archiver) em dupla com a Laura Arakaki. O problema envolvia a identificação e análise de e-mails duplicados presentes no dataset, que ocorrem porque muitos e-mails são enviados simultaneamente para múltiplas mailing lists, gerando réplicas em diferentes partições do Parquet. 

A tarefa proposta foi criar um script capaz de identificar essas duplicatas, agrupando os e-mails por `Message-ID` e hash do corpo (`body_sha1`), e permitir que o usuário defina quais listas seriam analisadas via variável de ambiente.

## Analisando com DuckDB

Para facilitar a análise foi utilizado o DuckDB. Para persistir as views e tabelas criadas durante a análise, é necessário inicializá-lo com um arquivo:

```bash
duckdb banco_duckdb.duckdb
```
Sem isso, ao utilizar apenas duckdb sem especificar um arquivo, tudo ficará armazenado apenas em memória temporária e será perdido ao encerrar a sessão.

No entanto, os arquivos Parquet estão distribuídos pelo projeto e organizados no formato Hive, onde a coluna list é inferida a partir dos nomes das pastas. Para manter essa classificação e facilitar as consultas, foi criada a seguinte view:

```sql
CREATE VIEW dataset AS 
SELECT * FROM read_parquet(
    '<caminho_base>/MLH-archiver/output/parser/dataset/**/*.parquet',
    hive_partitioning=true
);
```

## Codando o script no projeto

Após várias consultas utilizando o DuckDB foi decidido utilizar o seguinte script como base:

```sql
SELECT
	message_id,
	body_sha1,
	MIN(date) AS date,
	COUNT(*) AS number_of_replicas
FROM dataset
WHERE list='kernel.ime'
GROUP BY message_id, body_sha1
HAVING COUNT(*) > 1;
```

Com isso foi criado o arquivo `list_duplication.py`:

```python
list_duplication.py
import polars as pl
import os
import glob
 
 
def main(working_dir, output_dir):
    if not working_dir:
        print("Error: no input directory available.")
        return
 
    default_lists = "netdev,bpf,rust-for-linux"
    LISTS_OF_INTEREST = (
        os.environ.get("LISTS_OF_INTEREST") or 
        default_lists
    ).split(
        ","
    )
    LISTS_OF_INTEREST = [li for li in LISTS_OF_INTEREST if li]
 
    if not LISTS_OF_INTEREST:
        raw_dirs = glob.glob(f"{working_dir}/list=*")
        LISTS_OF_INTEREST = sorted(
            [os.path.basename(d).removeprefix("list=") for d in raw_dirs]
        )
        print(f"Using all available lists: {LISTS_OF_INTEREST}")
 
    for m_list in LISTS_OF_INTEREST:
        print(f"\nProcessing list: {m_list}")
 
        df = pl.read_parquet(f"{working_dir}/list={m_list}/*.parquet")
        # df = df.with_columns(pl.lit(m_list).alias("list"))

        duplicates = (
            df.group_by(["message_id", "body_sha1"])
            .agg([
                pl.min("date").alias("date"),
                pl.count().alias("number_of_replicas"),
                # pl.first("list").alias("list"),
            ])
            .filter(pl.col("number_of_replicas") > 1)
        )
 
        print(f"Found {len(duplicates)} duplicated messages in {m_list}")
        if len(duplicates) > 0:
            print(duplicates)
 
        output_path = os.path.join(output_dir, f"duplications_{m_list}.csv")
        duplicates.write_csv(output_path)
        print(f"Saved to {output_path}")
```
